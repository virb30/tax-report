# BUG-004: Broker creation returns `201`, then the frontend crashes inside `BrokersPage`

**Severity:** High
**Priority:** P1
**Type:** Crash
**Status:** Open

## Environment

- **Build:** `8979c62`
- **OS:** Linux
- **Browser:** Playwright CLI Chromium headless shell 149.0.7827.3
- **URL:** `http://127.0.0.1:4173`

## Summary

Creating a broker from the browser succeeds in the backend, but the frontend throws a runtime exception immediately after the `201 Created` response. The page logs a React error in `BrokersPage` and becomes unstable right after the create action.

## Reproduction

```bash
playwright-cli open --browser=chromium http://127.0.0.1:4173
playwright-cli click e15
playwright-cli fill e252 QA1
playwright-cli fill e254 "QA Broker"
playwright-cli fill e256 11.222.333/0001-44
playwright-cli click e258
```

Observed before the fix:

- `POST /api/brokers` returns `201 Created`.
- Browser console logs `TypeError: Cannot read properties of undefined (reading 'length')`.
- The React stack points to the `BrokersPage` / `EditBrokerModal` render path.

## Expected

After a successful broker creation, the page should show success feedback and reload the broker table without crashing.

## Root cause

`frontend/src/pages/brokers-page/use-broker-management.ts` expects `createBroker()` to return a `{ success, error }` shape. The live backend returns a direct broker payload on `201`, so `result.success` is `undefined`, the code falls into `setErrorMessage(result.error)`, and `errorMessage` becomes `undefined`. `frontend/src/pages/BrokersPage.tsx` then dereferences `brokerManagement.errorMessage.length`, causing the crash.

## Fix

Pending. Make the create/update/toggle broker frontend code consume the real backend response contract, or standardize the backend to the frontend's expected wrapper.
