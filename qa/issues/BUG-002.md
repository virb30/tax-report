# BUG-002: Import preview can require asset type resolution, but the UI exposes no way to resolve it before confirm

**Severity:** High
**Priority:** P0
**Type:** Functional
**Status:** Open

## Environment

- **Build:** `8979c62`
- **OS:** Linux
- **Browser:** Playwright CLI Chromium headless shell 149.0.7827.3
- **URL:** `http://127.0.0.1:4173`

## Summary

The browser import flow accepts a valid CSV, generates a preview, and enables `Confirmar importação`, but the confirm request fails with `422 Unprocessable Entity` when any supported row still has unresolved asset type. The UI only shows the error message and does not render any control to resolve the asset type before confirm.

## Reproduction

```bash
playwright-cli open --browser=chromium http://127.0.0.1:4173
playwright-cli upload /home/virb30/workspace/tax-report/qa/fixtures/ops.csv
playwright-cli click e70
playwright-cli click e71
```

Observed before the fix:

- Preview succeeds and shows `Conferência pronta: 1 transações.`
- Confirm fails and the page shows `Existem linhas suportadas sem tipo de ativo resolvido para: PETR4. Informe um tipo de ativo antes de confirmar.`
- No asset-type selector or repair affordance is rendered in the import workspace.

## Expected

If preview detects unresolved supported rows, the import UI must either:

- expose per-row asset-type resolution controls before confirm, or
- keep confirm disabled until the user resolves the missing data through an explicit browser path.

## Root cause

`frontend/src/pages/import-page/use-transaction-import.ts` hardcodes `assetTypeOverrides: []` in `confirmImport()`, while `frontend/src/pages/import-page/TransactionsPreviewTable.tsx` only renders a read-only preview table. The browser workflow has no UI for the unresolved-type branch that the backend enforces.

## Fix

Pending. Add an asset-type resolution step to the import UI and pass the selected overrides to `confirmImportTransactions`.

## Verification

- Confirm that unresolved rows can be fixed in the browser before confirm.
