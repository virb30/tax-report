# BUG-003: Initial balance page shows a contract error on load and does not refresh after a successful save

**Severity:** High
**Priority:** P0
**Type:** Functional
**Status:** Open

## Environment

- **Build:** `8979c62`
- **OS:** Linux
- **Browser:** Playwright CLI Chromium headless shell 149.0.7827.3
- **URL:** `http://127.0.0.1:4173`

## Summary

The `Saldo Inicial` workspace renders `Cannot read properties of undefined (reading 'message')` before any user action. After submitting a valid document, the backend persists the record with `200 OK`, but the frontend keeps the same error banner and does not refresh the saved-documents or positions tables.

## Reproduction

```bash
playwright-cli open --browser=chromium http://127.0.0.1:4173
playwright-cli click e10
playwright-cli fill e141 IVVB11
playwright-cli select e143 ETF
playwright-cli fill e145 "IVVB11 ETF"
playwright-cli fill e147 02.332.886/0001-04
playwright-cli fill e149 320
playwright-cli select e157 "XP Investimentos"
playwright-cli fill e159 5
playwright-cli click e163
```

Observed before the fix:

- The page shows `Cannot read properties of undefined (reading 'message')` on initial load.
- `POST /api/initial-balances` returns `200 OK`.
- `GET /api/initial-balances?year=2025` shows the saved document in the backend.
- The browser UI still shows the error banner and `Nenhum documento de saldo inicial salvo para 2025.`

## Expected

The page should load without a generic runtime error and should refresh documents and positions after a successful save.

## Root cause

`frontend/src/pages/initial-balance-page/use-initial-balance.ts` calls `unwrapApiResult(...)` for `listInitialBalanceDocuments()` and `listPositions()`, but those live endpoints return direct payloads like `{ items: [...] }`, not `ApiResult<T>`. The hook therefore tries to read `result.error.message` from a non-`ApiResult` response and leaves the UI in a broken post-save state.

## Fix

Pending. Align the frontend contracts for initial-balance and positions GET endpoints with the real backend response shape, or change the backend to return the documented `ApiResult<T>` wrapper consistently.
